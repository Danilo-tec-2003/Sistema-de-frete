Motor Fiscal

A API Go virou o nucleo tecnico da decisao fiscal

Ela concentra contrato, seguranca, motor de regras, calculo com decimal, persistencia e memoria de calculo.

Motor_fiscalGo + PostgreSQL

Contrato e responsabilidades

API HTTP

/health verifica disponibilidade.
/tax/preview calcula antes de salvar o frete.
/tax/simulate calcula e persiste auditoria.

Service Rules

TaxService valida e orquestra o fluxo.
FiscalRuleService aplica matching, prioridade e conflito.
TaxCalculator calcula ICMS, IBS e CBS com decimal.
Erros retornam JSON padronizado para o Java tratar.

Dados SQL

fiscal_rules guarda vigencia, CFOP, prioridade e status.
fiscal_rule_conditions decide quando a regra vale.
fiscal_rule_taxes guarda aliquotas por imposto.
fiscal_simulations registra memoria fiscal.

Por que Go aqui?

1Servico pequeno e focado
A API faz uma coisa bem definida: decisao fiscal.
2Contrato independente
O Java nao precisa conhecer classes internas do motor.
3Calculo previsivel
Valores trafegam como string e sao calculados com `decimal`, nao `float`.
4Regra versionavel
Vigencia, status, prioridade, fonte e revisao contabil ficam modelados.
5Auditoria nativa
Cada simulacao guarda regra aplicada, impostos e formula de calculo.
Escolha: contrato HTTP, regra no banco, calculo no service, auditoria persistida 7 / 9

Integracao finalizada

O Java chama o Go, recebe o calculo e grava o resumo no frete

A integracao foi pensada para nao expor token no navegador e para permitir rastreabilidade entre sistemas.

ContratoHTTP + JSON

sistema-fretes Java

Tela de frete solicita preview ou emissao.
`FreteBO` valida dados operacionais.
`TaxSimulationRequest.fromFrete` monta payload.
`MotorFiscalClient` envia JSON com headers.
`FreteDAO.atualizarResumoFiscal` grava retorno.

Motor Fiscal Go

Recebe `POST /api/v1/tax/simulate`.
Valida `X-API-Key`.
Propaga `X-Correlation-ID`.
Seleciona regra fiscal aplicavel.
Retorna CFOP, impostos, regra e memoria.

PostgreSQL Dados

Sistema Java: tabela `frete` guarda resumo fiscal.
Motor Fiscal: `fiscal_rules` guarda regras.
Motor Fiscal: `fiscal_simulations` guarda historico.
Motor Fiscal: `fiscal_simulation_tax_details` guarda memoria por imposto.
Seguranca: token fica no backend; rastreabilidade: correlation ID 8 / 9
